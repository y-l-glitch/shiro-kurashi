場所: デスクトップ\Claude\shiro-kurashi\
公開ページ: https://y-l-glitch.github.io/shiro-kurashi/
GitHubリポジトリ: https://github.com/y-l-glitch/shiro-kurashi


●更新は自動(基本なにもしなくてよい)

  毎月1日の朝9時に、タスクスケジューラ「レシピ検索の月次更新」が
  新着動画の取得→GitHubへの反映まで自動でやってくれる。
  (その時間にPCが起きていなければ、次にPCを起動したときに実行される)

  すぐ更新したいときだけ「更新してアップロード.bat」をダブルクリック。
  「完了！」と出たら成功。1〜2分後に公開ページに反映される。

  自動実行の記録は update-log.txt に残る。


●ホーム画面のアイコン画像を変えたいとき

  1. 新しい画像を用意する
     ・PNG形式・正方形(180×180ピクセル以上)
     ・ファイル名は必ず「apple-touch-icon.png」(半角英数)にする
  2. このフォルダ(shiro-kurashi)に上書きコピーする
  3. 「更新してアップロード.bat」をダブルクリック
  4. iPhone側で反映させる:
     ・ホーム画面の古いアイコンを長押し→削除
     ・数分待ってからSafariでページを開き直す
     ・共有ボタン→「ホーム画面に追加」

  ※iPhoneはアイコンを強く記憶するので「一度削除して追加し直す」のがポイント


●「字幕未取得」の動画について(基本ほうっておいてOK)

  YouTubeは1日にたくさん字幕を取りにいくと制限(429エラー)をかけてくる。
  1回の実行で取れるのはおよそ40本前後まで。特に「自動吹き替え動画」
  (字幕が英語の自動翻訳しかない動画)の翻訳字幕は制限が特に厳しい。
  ※制限に当たった動画も、タイトルと説明欄は索引済みなので検索はできる。
    レシピは説明欄に載っていることが多いので実用上は困らない。

  制限で取れなかった分は、通常の「更新してアップロード.bat」や毎月の
  自動更新のたびに自動でリトライされる(=放っておけば少しずつ埋まる)。
  なので基本は何もしなくてよい。

  すぐ埋めたい場合は、日を変えて「更新してアップロード.bat」を
  数日繰り返すのが確実(1日1回ペース)。
  一気に叩くと逆に制限が強まるので、まとめて何度も実行しないこと。

  ・新しく追加したチャンネル動画を、字幕は後回しにして今すぐ検索対象に
    加えたいとき(制限中でも可):
        python collect.py --meta-only
  ・「本当に字幕が無い」動画まで含めて全部を再チェックしたいとき(通常不要):
        python collect.py --retry-nosubs


●うまくいかないとき

  ・bat実行時にGitHubのログイン画面(ブラウザ)が出た
    →「Sign in with your browser」でログインすればOK。一度やれば覚えてくれる
  ・pushに失敗する
    →ネット接続を確認。直らなければClaudeに「shiro-kurashiのpushが失敗する」と相談
  ・手動でアップロードしたいとき(最終手段)
    →GitHubのリポジトリページ → Add file → Upload files に
      index.html をドラッグ&ドロップ → Commit changes


●各ファイルの役割

  更新してアップロード.bat … ワンクリック更新(これだけ使えばOK)
  collect.py             … 字幕収集スクリプト本体
  template.html          … ページの見た目の雛形(デザイン変更はここ)
  index.json             … 収集済みデータ(消すと全動画を取り直しになるので注意)
  index.html             … 完成品(自動生成されるので直接編集しない)
  apple-touch-icon.png   … ホーム画面用アイコン画像(自分で用意して置く)
